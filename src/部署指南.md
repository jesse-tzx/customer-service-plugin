# 飞猪客服工具箱 - 部署指南

## 快速开始

1. 把整个 `FliggyToolbox` 文件夹放到你本地
2. 打开 Chrome，访问 `chrome://extensions/`
3. 开启「开发者模式」
4. 点击「加载已解压的扩展程序」，选择 `FliggyToolbox` 文件夹

安装后，在飞猪客服 XP 页面右上角会看到 **🧰 工具箱** 按钮，点击即可打开工具列表。

---

## 工具箱架构

```
FliggyToolbox/
├── manifest.json     # 扩展清单
├── background.js     # 统一管理版本检查 + 统计上报
├── popup.html        # 工具箱主面板（工具列表）
├── popup.js          # 面板交互逻辑
├── tools.json        # 工具配置（添加新工具只需改这个文件）
├── server.js         # 统计服务端
├── content_xp.js     # XP 页面工具箱入口按钮
├── content_yida.js   # 宜搭表单自动填充
│
└── tools/            # 各个工具的独立页面
    ├── esu.html      # ESU 一键报备工具页面
    ├── esu.js        # ESU 工具逻辑
    └── (未来其他工具...)
```

## 如何添加新工具

你只需要做两件事：

### 1. 把新工具的代码文件夹发给我

把新工具的文件夹给我，我会帮你：
- 在 `tools/` 下创建对应的 html 和 js 文件
- 在 `tools.json` 中添加配置

### 2. 或者你自己配置 tools.json

在 `tools.json` 的 `tools` 数组中添加一项：

```json
{
  "id": "新工具id",
  "name": "新工具名称",
  "description": "工具描述",
  "icon": "🔧",
  "version": "1.0",
  "page": "tools/新工具id.html",
  "versionCheckUrl": "https://你的服务器/新工具版本.json",
  "enabled": true
}
```

## 统计服务端

### 启动服务

在 `FliggyToolbox` 文件夹目录下打开命令提示符（cmd），运行：

```
node server.js
```

启动后浏览器访问 `http://localhost:3000/stats.html` 即可查看统计面板。

### 统计面板内容

- **总用户数** — 有多少不同用户上报过
- **总打开次数** — 所有用户的工具打开次数之和
- **按用户明细** — 每个用户的具体使用记录
- **按日期趋势** — 每天的使用人数和次数

## 自动更新

每个工具的版本更新通过 `versionCheckUrl` 配置。在服务器上放一个 JSON 文件：

```json
{
  "version": "1.1",
  "date": "2026-05-18T10:00:00Z",
  "downloadUrl": "https://你的服务器/更新包.zip",
  "releaseNotes": "修复了XX问题"
}
```

工具列表中有更新的工具会显示红色「有更新」标签。

## 常见问题

| 问题 | 解决 |
|------|------|
| 看不到工具箱按钮 | 刷新页面 F5 |
| 统计面板打不开 | 先运行 `node server.js` |
| 新工具不显示 | 检查 `tools.json` 中 `enabled` 是否为 `true` |
| 提取数据失败 | 确保 XP 页面已完全加载 |
